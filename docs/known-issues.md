# 已知问题与待验证点

本文件只记录从当前仓库可直接观察到的问题，不把未验证推断写成事实。

## 已观察问题

### 1. 全局状态仍集中在 `AudioPlaybackConnector.h`

现状：

- 主窗口句柄、主 XAML Island、托盘状态、连接表、全局设置都仍在头文件中直接定义

影响：

- 模块间仍通过共享全局状态耦合
- 如果未来增加新的翻译单元并直接包含该头文件，可能产生重复定义问题

建议：

- 后续若继续演进，可逐步把这些状态收敛到应用上下文对象

### 2. 运行时模块虽然拆分，但仍以头文件内联实现为主

现状：

- `SettingsWindow.hpp`、`AutostartManager.hpp`、`ConnectionController.hpp` 等都以头文件实现为主

影响：

- 编译边界不强
- 模块职责已经拆开，但实现仍然和全局状态紧密绑定

建议：

- 如果后续继续扩展功能，可考虑逐步引入对应 `.cpp` 实现文件，降低耦合

### 3. 缺少自动化测试

现状：

- 仓库中没有单元测试、集成测试或 UI 自动化测试

影响：

- 自动连接、任务计划程序、自定义托盘交互都只能靠手工回归确认
- 结构性修改风险较高

建议：

- 后续若继续维护，可先补一份稳定的手工回归清单，再评估最小自动化验证

## 维护要求

- 问题消失后必须同步更新本文件
- 不确定的事项不要写成 bug
- 运行环境限制导致“暂未验证”的内容，应放进最终交付说明，而不是写成仓库缺陷
